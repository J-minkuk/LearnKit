# Interface vs Abstract

## Interface
* 모든 메소드가 추상 메소드여야 합니다.
* 기능의 재정의에 큰 의미를 두고 있습니다.
* implements 키워드를 사용해 구현 및 다중 상속이 가능합니다.
* 메소드의 껍데기만으로 구현을 강제함으로써 구현 객체의 같은 동작을 보장합니다.

## Abstract
* 하나 이상의 추상 메소드를 가지고 있습니다.
* 상속에 큰 의미를 두어 extends 키워드를 사용합니다.
* 다중 상속의 모호성 때문에 하나만 상속합니다.
* 자식 클래스는 반드시 추상 메소드를 재정의해야 합니다.
